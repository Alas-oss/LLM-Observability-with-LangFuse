# LLM Observability with LangFuse

## 1. Project Set-up overview
The ShopMate RAG pipeline architecture was migrated to utilize the **Groq AI Infrastructure Layer**, executing inference on the open-source **`openai/gpt-oss-120b`** foundation model. Full telemetry data ingestion is monitored using a Langfuse cloud tracking framework.

- Total Operational Traces Captured: 20
- Manucal Evaluations Logged: 5 `correctness` scores added via UI annotation (4 pass, 1 fail)
- Dataset Integration: 5 evaluations with constructed Q&A pairs under the `shop-mate` eval repository tracking index.

## 2. Perfromance & Latency Bottlenexk Brakedown

Based on lite telemetry data extracted from highest-latency trace cycle, the exact performance metrics include:
- Slowest Trace ID: `bc86218b-0a73-4b13-ae3f-8087303f6ceb`
- Total Trace Runtime (`rag_query`): 1.34 seconds
- Database Sub-Step (`retrieval`):  0.2 seconds
- Inference Sub-Step (`llm_call`): 1.32 seconds

Conclusion: The data above proves that `98.5%` of the application latency is caused entirely by the `llm_call` step. The local vector storage lookup overhead is practically negligible (0.02s). Because text generation scales linearly with token length, the 182-token completion block generated by the model remains the single biggest driver of user delay.

## 3. Engineering Insights

Using the console prints or basic logging statements only give use the string of our applications. LnagFuse revealed critical operation details that logs don't provide, like:

1. Token Inflation Detection: The dashboard showed the exact input-to-output conversion curve (ex: `111 input tokens -> 182 completion tokens`). In turn this allows us to notice if system prompts are unnecessarily bloated before we scale the data context.
2. Asynchronous Cache Separation: Langfuse’s prompt lifecycle tracker highlighted that our prompt cache eviction warning (`Prompt not found, evicting from cache`) was executing seamlessly via a structural fallback block without corrupting the operational runtime loop or risking live customer dropouts.
